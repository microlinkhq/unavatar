unavatar

Get unified user avatar from social networks (source code).

Usage

A daily rate limit is applied for preventing abusive usage.

Just query against https://unavatar.io/:input, being input:

  • an username: https://unavatar.io/kikobeats
  • an email: https://unavatar.io/sindresorhus@gmail.com
  • a site domain: https://unavatar.io/reddit.com

Query Parameters

Fallback

When it isn't possible to get a user avatar, a generic one is used. You can customize this behavior passing any external URL to be used, such as boringavatars.com:

e.g., https://unavatar.io/1337_user?fallback=https://source.boringavatars.com/marble/120/1337_user?colors=264653,2a9d8f,e9c46a,f4a261,e76f51

You can pass fallback=false for disabling it. In that case, a HTTP 404 Not Found status code will returned.

JSON

The avatars resolved can be embed directly in your HML markup.

Alternatively, you can consume the service as JSON passing json=true:

e.g., https://unavatar.io/kikobeats?json

Providers

The fastest service into resolve the query will be the avatar returned. You can also query for a specific service.
